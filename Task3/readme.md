# Требования к внешний интеграции

Компании PropDevelopment требуется взаимодействовать с внешним сервисом для управления домофоном и шлагбаумом командами и предоставление информации для автоматического открытия доступа. Чтобы не передавать во внешний сервис данные пользователей лучше организовать взаимодействие через запрос от внешнего сервиса с параметрами распознавания на предоставление доступа. 
Это исключит возможную утечку данных и внешней системе не нужно будет хранить данные пользователей. Так же данные распознавания должны шифроваться.

## Интеграция с системой "Интеллектуальный домофон"

Внешняя система "Интеллектуальный домофон". Созданием модели будет заниматься компания PropDevelopment. 

Для добавление нового человека в систему распознавания PropDevelopment заносит фото и данные человека в свою систему. Фото передается партнеру, который выполняет процедуру формирования лицевого шаблона и возвращает его в PropDevelopment для внесения в модель. 

Когда система "Интеллектуальный домофон" определяет человека по фото, то отправляет в шифрованном виде лицевой шаблон и информацию о помещении, куда он пытается зайти в систему PropDevelopment для запроса на разрешение доступа. 
PropDevelopment проверяет разрешен ему доступ в данное помещение или нет.


## Интеграция с системой "**Интеллектуальный шлагбаум**"

Внешняя система "Интеллектуальный шлагбаум". Созданием модели для системы будет заниматься компания PropDevelopment.

Модель на основании фото пытается расшифровать номер подъехавшего автомобиля. Если номер удается определить, то сервис партнера отправляет запрос в систему PropDevelopment, с токеном шлагбаума и данными номера в шифрованном виде. Система PropDevelopment проверяет разрешен ли доступ данному автомобилю и дает разрешение на открытие шлагбаума. 

## Требования к безопасности

- **Аутентификация JWT** (JSON Web Token). Все запросы от партнера и к партнеру должны содержать токен JWT для аутентификации партнера.
- **Авторизация OAuth 2**. Для взаимодействия с API партнером применяется метод авторизации с использованием OAuth 2. Это позволит контролировать методы, которые доступны партнеру;
- **Gateway**. Между API партнера и системой PropDevelopment дожен быть установлен шлюз который будет разрешать доступ только на выделенные порты, осуществлять фильтрацию трафика и экранировать систему PropDevelopment в закрытый сегмент; 
- **Шифрование трафика**. Весь трафик между системой PropDevelopment и API партнеров должен идти по протоколу HTTPS с использованием TLS1.3‑зашифрованного соединения. Сертификаты HTTPS должны выпускаться PropDevelopment или доверенными центрами.
- **Шифрование данных при хранении**. Вся конфиденциальная информация (ФИО, фото клиентов и собственников, фото машин, паспортные данные) должна храниться в зашифрованном виде в БД. 
- **Шифрование данных при передаче**. Передаваемые конфиденциальные данные (фото, токены, данные о помещениях) должны дополнительно шифроваться. Должен быть предусмотрен механизм смены паролей шифрования, когда системы должны уметь расшифровывать данные предыдущим паролем шифрования или уже новым. 
- **Обмен ключами шифрования**. Обмен ключами шифрования должен быть реализован не через каналы связи с API партнерами. Это должна быть интеграция через сторонний сервис, почту или аппаратный носитель.
- **Изоляция сети**. Обмен данными с системами партнеров должен происходить через брандмауер.
