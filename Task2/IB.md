### Основные компоненты системы

+ Клиентские сервисы: (client) 
    - работа с персональными данными: ФИО, номер телефона 

+ Cервисы для собственников: (tenant) 
    - работа с персональными данными: ФИО, номер телефона, список недвижимости, проведенные сделки
    - взимодействие с внешними источниками (гос органы, жкх)

+ Сервисы бухгалтерии
    - работа с финансовыми документами

+ DWH хранилище
    - единое хранилище всех данных

+ Auth cервисы
    - секретная информация, обеспечивающая безопасность передачи, хранения данных. 
    Обеспечивает интеграцию со сторонними сервисами на основе ключей доступа

### Основные риски и приоритеты

Организационные проблемы:
 - Множественные точки регистрации клиентов
 - Отсутствие единых стандартов обработки данных
 - Отсутствие системного контроля за передаваемыми данными

Технические недостатки:
 - Отсутствие единой системы мониторинга данных
 - Отсутствие технических ограничений на доступ к данным
 - Отсутствие механизмов аудита данных

Проблемы безопасности:
 - Контракты API содержат персональные данные
 - Внешние пользователи получают доступ к чужим данным
 - Жалобы пользователей о получении чужих данных
 - Риск утечки данных, Повышенная обеспокоенность клиентов безопасностью данных из-за новостей в СМИ
